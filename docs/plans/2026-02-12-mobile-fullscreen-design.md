# 移动端全屏复习模式设计

**日期**: 2026-02-12  
**状态**: 设计完成，待实现

## 问题

手机竖屏刷题时，开始复习后需要往下滑才能看到题目，顶部标题/统计信息占用太多空间。

## 目标

开始复习后进入全屏沉浸模式，最大化题目显示区域。

## 设计方案

### 全屏覆盖层

使用 `position: fixed` 覆盖层，点击开始复习后从底部滑入。

### 布局

```
┌─────────────────────────┐
│ ✕                  ●●○○○│  ← 左上X退出 + 右上进度点
├─────────────────────────┤
│                         │
│   5. Longest Palindro...│  ← 题目名（可折行）
│   MEDIUM · STRING       │  ← 难度 + 分类标签
│                         │
│   ┌─────────────────┐   │
│   │  ⚡ 模式识别      │   │  ← 练习卡片（全宽）
│   │                  │   │
│   │  核心算法是？     │   │
│   │                  │   │
│   │  A) 哈希表       │   │
│   │  B) 递归         │   │
│   │  C) 动态规划     │   │
│   │  D) 双指针       │   │
│   │                  │   │
│   └─────────────────┘   │
│                         │
└─────────────────────────┘
```

### 交互

| 操作 | 行为 |
|------|------|
| 点击 X | 退出全屏，返回主页 |
| 答完一题 | 自动切换下一题（0.5s 延迟） |
| 左右滑动 | 手动切换题目 |
| 完成所有题 | 显示总结 3s → 自动退出 |

### 进度指示器

- 位置：右上角
- 样式：小圆点
  - `●` 实心亮色 = 当前题
  - `●` 实心灰色 = 已完成
  - `○` 空心 = 未做

### 动画

- **进入**: `transform: translateY(100%)` → `translateY(0)`, 0.3s ease-out
- **退出**: 反向动画
- **题目切换**: 淡入淡出或左右滑动

## 实现要点

1. **HTML**: 新增 `#fullscreenQuiz` 容器
2. **CSS**: 全屏样式 + 动画 + 响应式
3. **JS**: 
   - `openFullscreen()` / `closeFullscreen()`
   - 修改 `smartRecommend()` 使用全屏模式
   - 添加滑动手势支持（touch events）
   - 自动切换逻辑

## 不做的事（YAGNI）

- 横屏适配（用户主要竖屏）
- 复杂手势（捏合缩放等）
- 主题切换（保持暗色）
